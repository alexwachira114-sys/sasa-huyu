For sasa-huyu on Vercel, the cause is clear: Replit injects registry=http://package-firewall.replit.local into that project's .npmrc, and Vercel's build servers can't reach Replit's internal network. There are two ways to fix it inside the sasa-huyu repo:

Option A — override in vercel.json (quickest, no .npmrc changes)

Add a vercel.json at the project root:

{
  "installCommand": "npm install --registry https://registry.npmjs.org"
}

Option B — fix .npmrc directly

Open sasa-huyu/.npmrc and replace:

registry=http://package-firewall.replit.local

with:

registry=https://registry.npmjs.org/

Then commit and push. Vercel will pick it up on the next deploy.

Option A is safer because it doesn't break the Replit development environment (which needs the local registry to work). The vercel.json override only applies during Vercel builds.
