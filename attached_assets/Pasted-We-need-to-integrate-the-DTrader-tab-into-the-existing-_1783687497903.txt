The goal is NOT to maintain legacy authentication compatibility.

The goal is:

"Make the DTrader tab behave exactly like the other existing tabs in this project by wiring it into the existing PKCE authentication flow."

* User authentication is handled by the existing PKCE flow.
* After login, the project already has the authenticated user session.
* Existing tabs are successfully using this new authentication.
* DTrader is being added as another tab inside the same application.

* old authToken flow
