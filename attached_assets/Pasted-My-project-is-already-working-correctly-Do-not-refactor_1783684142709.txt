Current Problem

The DTrader iframe is being loaded with:

acct1=<loginId>
token1=<token>

The iframe then performs a Deriv WebSocket

authorize(token1)

request.

For legacy users this works because token1 is a real Deriv WebSocket API token:

a1-xxxxxxxxxxxxxxxx

For PKCE users the code passes:

NEW_AUTH_token

which is an OAuth JWT bearer token.

The Deriv WebSocket API rejects this immediately with:

Input validation failed: authorize

because WebSocket authorize does not accept OAuth bearer tokens.

Important

Do NOT attempt to fix this by:

passing NEW_AUTH_token directly
storing loginId as token
falling back to invalid tokens
mixing legacy and PKCE logic

Those approaches have already been tested and do not work.
